# 附录 B：事件检测方法迭代记录

## B.1 TKEO（第一代，已废弃）

**实现**：`emg/event_detection.py.bak`（已归档至 `backup/`）

**核心算法**：
1. TKEO 对每个通道的滤波后信号计算能量算子
2. 双阈值状态机（OnSigma=15, OffSigma=5）检测激活边界
3. 自适应分段合并（`split_long`）+ 碎片合并（`merge_fragments`）
4. 双通道协同验证（重叠判定）

**实验评估**：
- 在训练集上能检测 4–6 个周期
- 但对特定文件（如 `emg_05_qpj_1`）检测偏多或偏少
- 硬编码参数无法适应不同受试者的肌电特性

**废弃原因**：
- 12 个内部硬编码参数：`BaselinePct=20, OnSigma=15, OffSigma=5, MaxDur=3.0, MinDur=0.15...`
- labels.csv 由 Top-K 寻峰法生成，TKEO 检测的边界与之不一致
- 对噪声敏感（TKEO 放大高频噪声）[Solnik et al., 2010 已指出此缺陷]

## B.2 Top-K 寻峰（第二代，已优化）

移植自 `dataset/quality_rules.py:detect_cycles()`。

**核心算法**：
1. Hilbert 包络 + 6Hz 低通平滑
2. scipy peak_prominences 寻峰 + 自适应 K 值
3. 相邻峰值间谷底为边界

**问题**：全局操作，不区分活动/静息段。静息基线无法统一。

## B.3 两阶段自适应（第三代，当前版本 v2.0）

**阶段 1：活动段检测**
- RMS 融合包络（替代 Hilbert，200ms 窗，50% 重叠）
- 基线自动估计（下 20% 分位数 + MAD 噪声估计）
- 自适应阈值：`baseline + activity_sigma × sigma`
- 合并相邻活动段（间隔 < 0.3s）

**阶段 2：混合边界周期分割**
- 首尾：阈值交叉（激活/放松的生理起点）
- 中间：峰间谷底法（处理紧密动作）

**关键参数**：
- `activity_sigma`（默认 3.0，用户可调 0.5–15）：控制活动检测的灵敏度
- 越小 → 更敏感 → 更多活动段被检测
- 越大 → 更保守 → 仅显著激活被标记

**实际影响**：
- Group03/12.mat：使用前（v1.0）0 周期 → 使用后 6 周期

## B.4 事件检测验证

检测结果与 labels_v2.csv 标签的匹配率：在 15 个受试者的 65 个文件中，所有具备 >3 个标准样本的受试者均获得 ≥ 90% 的周期匹配率。
