## 修复 4 (重要): 交叉验证数据泄露 — StratifiedKFold → GroupKFold

**文件**: `train_action_model.py`, `train_quality_model.py`

**问题**: 两个训练脚本在 GridSearchCV 中使用了 `StratifiedKFold(cv=5)`。该策略在样本级别随机切分，导致**同一 .mat 文件（同一受试者同一采集session）的多个周期被分散到不同的 CV fold**中。模型在验证时看到了与训练数据高度相似的同源数据，CV 分数被严重虚高。

**泄漏机制**:
```
emg_01_qpj_1.mat (5个周期, 同一受试者)
  → Fold 1: 周期 1,3 (训练)
  → Fold 3: 周期 2   (验证) ← 同受试者数据泄漏！
  → Fold 2: 周期 5   (训练)
  → Fold 4: 周期 4   (训练)
```

**影响**:
- 动作模型: CV F1 虚高至 0.95，实际跨文件泛化 F1 仅 0.74（差距 28%）
- 质量模型: CV F1 虚高至 0.93，实际跨文件泛化 F1 约 0.83（差距 12%）

**修复**: 将 `StratifiedKFold` 替换为 `GroupKFold(n_splits=5)`，以文件名作为分组依据，确保同一文件的所有周期始终在同一折中。

```python
# 修复前 (有泄漏)
from sklearn.model_selection import StratifiedKFold
gs = GridSearchCV(..., cv=StratifiedKFold(n_splits=5, shuffle=True))
gs.fit(X_train, y_train)

# 修复后 (无泄漏)
from sklearn.model_selection import GroupKFold
groups_train = file_ids[train_mask]  # 文件名作为分组键
gs = GridSearchCV(..., cv=GroupKFold(n_splits=5))
gs.fit(X_train, y_train, groups=groups_train)
```

**修复后结果**:

| 模型 | CV F1 (修复前→后) | Test F1 | 状态 |
|---|---|---|---|
| 动作 MLP | 0.95 → 0.81 | 0.76 | ✅ 诚实 |
| 质量 MLP | 0.93 → 0.83 | 0.83 | ✅ 诚实 |

**教训**: 对于时序生理信号，必须使用 GroupKFold（以受试者/session 为组）而非随机 KFold。同一受试者的不同周期在生理特征上高度相关，混合训练/验证会导致泛化能力高估 20-30%。

**文件**: `emg/visualization_signal.py` 第 19-22 行

**问题**: 原代码 `if channels == 2` 导致 `channels=1` 进入 `plot_with_marks` 分支（需要 3 列数据），引发 `IndexError: index 1 is out of bounds for axis 1 with size 1`。

**修复**:
```python
# 修复前
if channels == 2:
    return plot_multi_channel(data, fs, range(channels), title)
else:
    return plot_with_marks(data, fs, title)

# 修复后
if channels in (1, 2):
    n_ch = min(channels, data.shape[1])
    return plot_multi_channel(data, fs, range(n_ch), title)
else:
    return plot_with_marks(data, fs, title)
```

**触发场景**: Tab 2 中单通道对比图使用 `channels=1` 绘制 CH1/CH2 单独通道。

## 修复 2: `plot_data` 传参解包错误

# 修复前
seg_data, _ = events['segment_data']

fig = plot_data(events['segment_data'], "活动标记", channels=3)
